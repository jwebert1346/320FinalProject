Introduction:¶

Financial crimes, including money laundering, tax evasion, and offshore transfers pose significant threats to global economies. Detecting and understanding the patterns behind these illicit transactions is a crucial step in mitigating their impact. This Global Black Money Transactions Dataset, which we procured from Kaggle.com (https://www.kaggle.com/datasets/waqi786/global-black-money-transactions-dataset), provides a rare opportunity to delve into the intricate web of financial irregularities.

Our project aims to analyze this dataset to address key questions such as:

What are the prominent patterns in illicit financial transactions across countries and industries? How does the Money Laundering Risk Score correlate with transaction attributes like amount, destination country, and involvement of shell companies? What trends emerge regarding the reporting of suspicious transactions by authorities? Answering these questions is essential because financial crimes have far-reaching consequences, including undermining public trust in institutions, draining resources from critical sectors, and exacerbating inequality. Through this analysis, we hope to uncover actionable insights that could aid in developing more robust anti-money laundering (AML) algorithms and policy interventions.

For our initial analysis, we will try to find any features in our data which could be correlated. Finding correlations will be useful for later when we try to create an AI model to make predictions. Seeing how AI predictions are made almost entirely based on correlations between data, finding correlations in our current data will give us information about which features to use and what label we may want to try to predict when we go about writing our AI models.

We start by investigating any correlations between the type of a transaction and the Tax Haven Country. Different countries may have different laws relating to types of transactions, so some types of fraud may be easier to commit using specific countries as a Tax Haven Country. To find out if this correlation does in fact exist, we shall use a Chi-Squared test between these columns.

In [ ]:
# 1:
# is there a significant association between the Transaction Type and the Tax Haven Country (chi square)
#H0 = There is no sigificant association between Transaction Type and the Tax Haven Country
#H1 = There is a significant association between Transaction Type and the Tax Haven Country
# alpha = .05

# We perform a chi square test here because we are determining the association between 2 categorical variables.

contingency_table = pd.crosstab(df['Transaction Type'], df['Tax Haven Country'])

Chi-Square Statistic: 41.26891811614076

Based on the results of the chi square test, as well as our alpha value of 0.05, we reject the null hypothesis and accept the alternative, that there is a significant difference between Transaction Type and Tax Haven Country (p < .05).

The Chi Squared test gave us a p-value of 0.0034, which is less than the standard alpha value of 0.05. Since our p-value was less than our alpha value, we can reject the null hypothesis and accept the alternative hypothesis. So, we can conclude that there is a significant difference between the Transaction Type and Tax Haven Country.

Primary Analysis¶

We decided to investigate the link between details of a transaction and the legality of said transaction. Since we are trying to predict a categorical label of some data, based on other features, we decided that this problem would be perfect for a classifier model. For our AI model, we picked a Random Forest Classifier, since Random Forest Classifiers can easily handle categorical data, and unlike a decision tree, a Random Forest is less prone to overfitting and biases.

To analyze our data and answer the questions we put forth in our introduction, we decided to train a machine learning model to perform a classification task on our dataset. We decided to use the features "Tax Haven Country", "Industry", and "Transaction Type" to predict "Source of Money", which evaluated to Legal or Illegal. Our reasoning for this, as explained earlier from our data exploration section, was that these features had the highest correlation with "Source of Money". Many of the features in our dataset had little to no correlation with the legality of the transaction, so we decided to remove those features. By removing features from the dataset, we attempt to remove any noise from our AI model's predictions. This should make our Random Forest Classifier better at classifying each transaction's legality.

First, we'll do some additional preprocessing before putting the data into the model to allow it to fit the demands of the random forest tree.

def random_forest_classifier(final_cols, current, new_df, seed = 40, test_size = 0.2, k_folds = 5):
  X = new_df[final_cols] # drop prediction target and non-normalized data
  y = new_df[current] # prediction target


  X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=test_size, random_state=seed)


  skf = StratifiedKFold(n_splits=k_folds, shuffle=True, random_state=seed)

  model = RandomForestClassifier(n_estimators=100, random_state=seed) # default is 100, can be upped for more accuracy but on small data sets that wont work so idk about ours
  model_name = "Random Forest"

  model.fit(X_train, y_train)
  y_pred = model.predict(X_test)
  cm = confusion_matrix(y_test, y_pred)

  plt.figure(figsize=(6, 5))
  sns.heatmap(cm, annot=True, fmt='d', cmap='Blues', xticklabels=model.classes_, yticklabels=model.classes_)
  plt.title('Confusion Matrix - Random Forest Classifier')
  plt.xlabel('Predicted')
  plt.ylabel('Actual')
  plt.show()

  np.random.seed(seed)
  scores = cross_val_score(model, X_train, y_train, cv=skf)
  print(f"Model: {model_name}")
  print(f"Accuracy: {scores.mean()} (+/- {scores.std()})")
  return X_train, X_test, y_train, y_test, y

result = random_forest_classifier(['Tax Haven Country', 'Industry', 'Transaction Type'], 'Source of Money', df)

We have discovered that we can use the features "Tax Haven Country", "Industry", and "Transaction Type" to predict whether or not a transaction is legal.
